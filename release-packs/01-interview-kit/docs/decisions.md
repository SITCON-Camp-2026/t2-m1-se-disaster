# 需求取捨決策

> 這份文件可以由 Codex 先產生草稿，但最後必須由你刪除、修改或補充。請不要讓 AI 替你做最後決策。

## 我決定 v1 優先服務的使用者

請勾選一個主要對象，也可以補充原因。

- [ ] 回報者
- [x] 資訊整理者
- [ ] 行動者

原因：作為資訊整理者可以快速先分類回報者在複雜情況下迅速寫出的亂資料，再給行動者明確的動作指示。

## 這個使用者最想完成什麼

- 快速的整理並分類回報者給出的雜亂資訊
- 給行動者明確的指示要做什麼

## 這個使用者最怕什麼錯誤

- 回報者給的資訊不夠明確，或分類種類不夠多，讓資訊整理者難以分辨並整理大量資訊

## 我今天暫時不處理的需求

- 更多樣的救災類型分類
- 更多樣的救災需求分類
- tags
- 提示詞i
- 回報者種類

## 我不採用的 AI 建議

請至少寫一項，並說明原因。

- AI 建議：沒有確切來源人名回報
  - 不採用原因：有些人不喜歡洩漏自己個資，因此仍有匿名回報的必要（隱私保護的需求）

## 必須保留人工確認的判斷

- 過低的或過高的極端尋認分或完整度的資料，因為AI可能判斷錯誤。
